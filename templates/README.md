# Templates

Reusable templates for Novel Framework projects.

## Purpose

These templates provide standardised starting points for the artifacts used in the Novel Framework workflow. They are framework-level tools and must not contain case-study canon. Each template includes placeholders, examples, and a validation checklist.

## Available Templates

| Template | When to Use |
|---|---|
| `prose-probe.md` | Testing tone and voice for a single scene before writing a full chapter packet. |
| `prose-probe-review.md` | Reviewing a prose probe for voice, characterisation, and compliance with the style guide. |
| `chapter-packet.md` | Translating architecture and probe findings into a constrained blueprint for prose drafting. |
| `chapter-packet-review.md` | Reviewing a chapter packet for completeness, over-prescription, and canon governance. |
| `acceptance-gate-checklist.md` | Validating any artifact before accepting it as complete. |

For the workflow that connects these templates, see `docs/workflow.md`.

## Usage Notes

- All templates are framework-level. Do not embed case-study canon in them.
- Each template includes a validation checklist. Run it before declaring the artifact complete.
- Templates are starting points. Adjust structure to fit the project's needs, but preserve the non-canon notice and canon-governance sections.
- Use repo-relative paths when referencing other files.
